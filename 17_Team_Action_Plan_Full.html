销售增长团队行动计划全量文档

销售增长团队只负责一件事：把战略白皮书转化为可成交、可复购、可证明的收入系统。市场负责声量和素材，销售负责客户判断、Pipeline 推进、成交、增购和转介绍。

Convert
把线索变成有效商机
Close
把诊断变成 Agent OS 订阅
Attach
把订阅延展到 FastPay / TermPay / MoreFund
Refer
把满意客户变成转介绍来源

战略锚点与销售解释

白皮书锚点销售侧解释客户听得懂的话
Agent OS for Platform-Native SMEs不是卖单点工具，而是卖跨境卖家的经营智能体系统。让系统每天帮你看现金流、库存、广告、回款和风险。
Six Agents + OrchestratorInsight、Deal、Spend、Capital、Treasury、Sentinel 分别对应经营判断、销售增长、费用管理、资金方案、资金调度、风险预警。你不需要一次买全，我们先从最痛的环节开始。
FastPay / TermPay / MoreFundTrio 是智能体判断后的资金执行层。看清问题以后，系统帮你匹配可执行的回款和资金方案。
L1-L4 商业模型按客户规模、复杂度、付费意愿和资金需求推荐不同级别。小卖家买简单判断，大卖家买全流程管理。
First Close / Second Close / Amazon PR销售要为融资节点提供真实收入和客户证据。每一次成交都要能沉淀为可审计的增长证据。

12 周行动计划

W1-W2
销售语言与客户分层定稿

统一 Agent OS 话术、L1-L4 判断标准、CRM 必填字段、合规边界和首批销售脚本。

W3-W4
Pipeline 与 CRM 运行

上线 Lead Scoring、客户诊断记录、下一步动作、商机阶段、成交概率和团队周会机制。

W5-W6
Demo 与试用转化

围绕现金流诊断、Agent 推荐、FastPay/TermPay/MoreFund 方案，跑通从试用到订阅的路径。

W7-W8
Amazon 节点获客准备

配合市场和业务管理团队准备 Amazon PR 后的线索承接、SLA、分层转化和销售演示材料。

W9-W10
Attach 与 Referral

从已订阅客户中识别资金产品 attach 机会，建立成功案例、转介绍触发点和客户健康分。

W11-W12
复盘与 Second Close 证据包

输出 ARR、Pipeline、转化率、客户案例、产品反馈、风险清单和下一阶段增长假设。

Amazon PR 后 24 小时 SLA

0-2h
线索入库与分配

所有 Amazon PR 线索进入 CRM，自动标记来源、平台、国家、GMV 区间和优先级，分配到 SDR / AE。

2-6h
首次触达

完成电话、邮件或微信首触。话术以经营诊断开场，不以额度和费率开场。

6-12h
客户分层与下一步

完成 L1-L4 分层、主痛点记录、推荐 Agent 和下一步动作。

12-24h
预约 Discovery / Demo / 3 分钟测算

L1/L2 约诊断会议，L3 约 Trial，L4 进入 3 分钟 FastPay Starter。未预约原因必须写入 CRM。

额度 / 费率 / 审批合规话术锁定

不得说必须改成使用场景
“你这个客户肯定能批。”“我们可以先做准入评估，结果需要结合平台数据、回款记录、资方规则和风控审核。”客户询问能否通过。
“额度大概能给到多少。”“初步测算可以作为参考，最终可用额度以完整材料和资方审批结果为准。”客户询问额度。
“费率就是 X%。”“费用会根据产品类型、期限、风险评估和资方规则确定，我们可以在评估后给你正式方案。”客户询问费率。
“最快一定今天到账。”“我们会尽快推进材料和审核流程，实际时间取决于资料完整度、资方审核和回款路径。”客户催放款。
“你先买 Agent OS，就一定更容易拿钱。”“Agent OS 能帮助你整理经营数据和资金需求，但融资准入仍需独立评估。”Agent OS 与金融产品联动。

跨团队协作机制

销售与市场

市场提供线索来源、活动、物料和品牌背书；销售反馈真实客户问题、成交话术、异议和案例。

销售与产品

销售把客户最关心的诊断输出、Agent 推荐逻辑和 Demo 反馈回传，推动产品优先做可成交场景。

销售与技术

技术支持 CRM 字段、AI 分析、数据接入和 Demo 稳定性，确保销售现场能演示。

销售与财务/合规

共同确认价格、回款、额度表达、风险提示和资方准入边界，避免过度承诺。
