由 AI 产出的用户旅程：首次接触 → 付费 → 留存

这个页面展示最终对客形态：客户看到的是清晰的经营诊断、可执行建议和资金方案；销售看到的是 CRM 内的客户分层、推荐 Agent、下一步动作和成交概率。

L4

小体量卖家

需要快速判断是否适合 FastPay 或轻量 Agent。

L3

成长卖家

需要 1-2 个 Agent 管理现金流、广告或库存。

L2

规模卖家

需要组合 Agent + Trio 产品提升资金效率。

L1

头部卖家

需要 Agent OS、定制资金方案和风控协同。

全流程用户旅程

1. 首次接触
Referral / 活动 / Amazon 节点 / 银行伙伴
用户看到

“3 分钟经营诊断”或“现金流压力测试”。

用户动作

填写店铺、GMV、平台、主要痛点。

AI 输出

初步分层 L1-L4、可能痛点和推荐入口。

销售跟进

24 小时内发起 Discovery，CRM 生成首个任务。

2. 资格判断
Lead Scoring
用户看到

可视化评分：增长机会、资金缺口、风险等级。

用户动作

补充回款周期、库存压力、广告预算。

AI 输出

推荐 Agent：Insight / Spend / Capital / Sentinel。

销售跟进

判断是否进入 SQL，并确定第一次会议目标。

3. Discovery
电话 / 会议 / 微信
用户看到

不是销售推销，而是基于经营问题的提问。

用户动作

确认当前最痛问题：缺现金、压库存、广告 ROI 低、回款慢。

AI 输出

生成客户画像、痛点排序、购买阻力和话术建议。

销售跟进

选择 Demo 场景，并在 CRM 写入下一步。

4. Demo / 诊断
Agent OS Preview
用户看到

现金流预测、库存资金占用、回款建议、风险预警。

用户动作

选择要先解决的 1-2 个问题。

AI 输出

生成 Agent 组合、Trio 推荐和 ROI 假设。

销售跟进

推进试用或订阅方案，不在 Demo 阶段过度承诺额度。

5. 数据授权 / 试用
SP-API / CSV / 轻量表单
用户看到

授权说明、数据使用边界、试用目标。

用户动作

授权或上传必要数据，确认试用周期。

AI 输出

72 小时内给出第一份可用诊断。

销售跟进

用诊断结果推进付费，不用泛泛功能介绍。

6. 付费方案
Subscribe
用户看到

推荐级别、Agent 列表、Trio 产品建议、预期价值。

用户动作

确认购买级别、合同、付款和启用时间。

AI 输出

生成 Proposal、实施任务和成功标准。

销售跟进

锁定 Close Plan，并同步 AM/CSM。

7. 激活与留存
Activation / Retention
用户看到

每周经营提醒、资金建议、异常预警、月度复盘。

用户动作

查看建议、申请产品、反馈结果、邀请团队成员。

AI 输出

健康分、续费风险、增购机会、Referral 时机。

销售跟进

推动续费、增购和转介绍，形成客户案例。

不同客户的购买判断

客户类型最可能购买判断依据销售动作
L4 小体量轻量诊断 / FastPay 入口GMV 小、痛点单一、预算敏感。降低理解成本，先证明一个结果。
L3 成长型单 Agent + FastPay增长快但现金流、库存或广告有明显短板。用一个高痛点场景成交。
L2 规模型组合 Agent + Trio多店铺、多 SKU、多资金需求。用 ROI 和团队协作价值推进。
L1 头部Agent OS + 定制方案组织复杂、风控要求高、资金需求大。以方案销售和高层对齐为主。

CRM 数据回写

每一次用户交互都要回写 CRM：客户分层、主痛点、推荐 Agent、推荐 Trio 产品、授权状态、试用结果、下一步、成交概率、合规风险、续费风险、Referral 潜力。这样销售团队不是靠记忆推进客户，而是靠 AI + CRM 的共同判断推进成交。
