SA 销售团队执行计划 v1.3

本计划只解决一个问题：SA 团队如何在 SME 中小客户战场，把 Agent OS 战略落成可运营的获客、诊断、试用、订阅、金融场景触发、续费和转介绍闭环。Customer Journey Demo 暂停，本文件优先作为 12 周执行与周会管理依据。

01 · Role Boundary

先明确：SA 不是泛销售，而是 SME 批量转化团队

v1.3 将 SA 定义为 Sales Acquisition / Sales Action，负责一线销售、L1-L4 作战、Demo 承接、转化、Referral。但结合会议输入和团队现实，SA 的主战场必须收敛到 SME：L4 批量、L3 核心、L2 选择性推进、L1/复杂 L2 转交。

主攻

L4 中小卖家

月 GMV $10K-$50K，老板直接决策，适合 3 分钟诊断、Free hook、批量试用、快速订阅。

核心

L3 成长 SME

月 GMV $50K-$150K，有广告、库存、回款错配，适合 14 天 Trial 和 Pro 订阅。

选择性

L2 上沿 SME

月 GMV $150K-$300K，只有当采购、账期、备货、回款问题清晰且老板可参与时推进。

转交

L1 / 复杂 L2

多站点、多币种、CFO/银行协同复杂，SA 完成识别与资料收集后转 SC / KA / CGO。

03 · ICP & Pain Map

目标客户画像：什么样的跨境卖家会需要 Agent OS

客户不是因为“可以授权数据”才需要 Agent OS，而是因为生意增长到某个阶段后，人、钱、货、广告、回款和供应商节奏开始互相牵制。数据只是诊断燃料，真正的销售切入点是经营压力。

增长卡住

有销售额，但不知道下一步怎么增长

SKU 多了、广告花了、店铺有订单，但老板不知道该推哪个品、停哪个品、加多少预算。

资金卡住

账上有应收，但现金流跟不上节奏

平台回款、广告投放、补货付款和供应商账期不同步，旺季一来就容易断档。

管理卡住

多平台多环节，靠人盯已经不够

老板、运营、财务各看一套表，库存、广告、回款、利润和风险无法每天合到一起判断。

决策卡住

不是缺数据，而是缺可执行建议

客户能看到报表，但看不出优先级；需要 Agent 把经营信号翻译成下一步动作。

卖家状态真实痛点数据能帮助判断什么推荐 Agent销售切入点
有稳定订单，但增长变慢老板想增长，但不知道该从 SKU、类目、广告还是库存先下手。GMV、订单、SKU、类目、转化和库存节奏。Insight Agent用“下一步增长机会诊断”引起兴趣。
广告投入越来越高预算加了，但利润没有同步变好，运营说不清钱浪费在哪里。广告花费、ROAS、转化、库存覆盖、销售节奏。Spend Agent用“广告和费用效率诊断”切入。
回款和备货节奏不匹配平台回款未到，补货、广告和账单先到，旺季更明显。回款周期、应收、库存、采购和广告计划。Capital Agent用“现金流时间表”说明是否进入回款提速或备货支持评估。
供应商付款压力大订单有了，但订金、尾款、账期短，导致现金被采购吃掉。采购订单、供应商付款、账期、销售预测。Deal Agent用“采购付款节奏诊断”切入供应商账期服务评估。
多平台/多账户经营钱、库存、币种、店铺表现分散在不同系统，老板无法快速判断。平台余额、回款路径、币种、账户和店铺表现。Treasury Agent用“资金和账户效率看板”切入，复杂客户转客户成功深化。
店铺波动或资料不完整退款、异常、准入资料、店铺健康影响推进速度和金融服务边界。店铺健康、退款、异常波动、资料完整度。Sentinel Agent用“风险预警和准入准备”保护客户，也保护销售不乱承诺。
销售判断原则：先判断客户有没有“增长、资金、管理、决策”四类压力，再判断能不能用数据做诊断。数据授权是 Trial 和深度分析的必要条件，但不是销售主卖点。对客户只讲经营价值：更快看清问题、更少靠人盯表、更早发现资金和风险压力。

05 · Lead Generation

获客不能只靠销售冷打，必须和市场、渠道一起做

豆沙包品牌力还不足以自然覆盖全量卖家，SA 的线索目标必须接近真实：以存量客户、渠道转介绍、服务商、社群活动、市场内容和平台合作为主，冷启动名单只作为补充。

06 · Operating Path

SA 的实际销售路径：一个入口、两个轨道、六个阶段

SA 不把客户拉进复杂战略里。先用一个低门槛经营诊断建立信任，再按客户痛点进入 Agent Trial、订阅或资金场景评估。带金融性能的 Agent OS 是 SME 必卖主线，不是可有可无的附加项。

01 · MQL

来源：官网、Amazon PR、存量客户、服务商、社群、Referral。24 小时内首触。

02 · SQL

确认平台、GMV、主痛点、决策人、是否 SME 主战场。

03 · Free Check

L4 做 3 分钟诊断；L3 做 14 天 Trial 预约；L2 做资金节奏图。

04 · Demo / Trial

展示 Agent 输出，不展示概念。客户必须看到至少 1 条可执行建议。

05 · Paid / 经营资金场景评估

成交 Agent OS 订阅；Capital / Deal / Spend 中触发回款、账期、备货评估。

06 · Renew / Refer

周报复盘、ROI Proof、续费风险、转介绍名单。

轨道 A · 带金融性能 Agent OS
Capital Agent 必卖回款/备货判断，Deal Agent 必卖账期/供应商判断，Spend Agent 必卖广告/库存资金效率判断。金融产品只在 Agent 输出后进入评估。
轨道 B · Pure Agent OS 订阅
Insight / Spend / Sentinel 可独立成交给不需要融资的客户，用来证明 Agent OS 有独立订阅价值。
Agent OS 性能销售必须卖的价值触发场景后续金融服务销售边界
Capital Agent判断回款、备货、广告、采购之间的资金错配。平台回款未到账、旺季备货、爆品加单、广告放量。回款提速 / 备货支持评估。不承诺额度、费率、审批。
Deal Agent判断供应商账期、采购节奏、订单付款是否拖累现金流。供应商要先款、账期短、采购订单集中。账期安排 / 供应商付款评估。不替客户签合同，不承诺账期结果。
Spend Agent判断广告、库存、物流和平台费用是否消耗利润。ROAS 下降、库存周转慢、广告预算上涨。资金效率优化后再判断是否需要备货支持。不自动调整广告预算。
Insight Agent判断下一个增长点、SKU 机会和类目趋势。客户想增长但不知道投哪个 SKU / 类目。如增长机会明确，再转 Capital 判断备货资金。不承诺 GMV 增长。
Sentinel Agent判断店铺健康、资料完整、风险边界。店铺异常、退货波动、资料不完整。所有金融服务前置风控边界。不绕过风控和人工复核。
Treasury Agent判断多币种、账户余额、FX 和资金调度。多平台、多币种、多账户客户。成熟客户转客户成功 / KA 深化。SA 识别后转交，不强行复杂成交。

CRM 是 SA 的作战系统，不是客户看的产品页面
