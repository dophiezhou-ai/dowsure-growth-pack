销售增长团队行动计划总结 1-pager

一句话：12 周内把战略白皮书转化为销售可执行的 Agent OS 订阅增长系统，并沉淀可用于融资、Amazon 节点和团队复制的增长证据。

目标

让销售团队能够清楚判断客户类型、推荐合适 Agent OS 级别、推进订阅成交，并通过 FastPay / TermPay / MoreFund 做增购和客户价值证明。

时间线

W1-W2：销售语言、客户分层、CRM 字段。

W3-W4：Lead Scoring、Pipeline、CRM 运行。

W5-W6：Demo、诊断、试用、订阅。

W7-W8：Amazon 节点线索承接。

W9-W10：Attach、案例、Referral。

W11-W12：复盘与 DD Evidence Pack。

主要活动

  • 用 L1-L4 判断客户购买级别。
  • 用 Discovery 找出现金流、库存、广告、回款、风险中的主痛点。
  • 用 CRM 记录推荐 Agent、推荐产品、下一步和成交概率。
  • 用 Demo 展示诊断结果，而不是只讲概念。
  • 用转介绍和客户健康分推动续费与增购。

产出

  • 销售 5 分钟上手卡。
  • 销售话术与作战手册。
  • 销售智能 CRM Demo。
  • AI 用户旅程全流程页面。
  • Pipeline Dashboard 与增长证据包。
  • SG-SKU / Owner / Date 交付管理表。
  • Amazon PR 后 24h SLA Playbook。
  • 额度 / 费率 / 审批合规话术锁定版。

周度硬指标

Revenue：每周新增 ARR、累计 ARR、订阅客户数。

战略证明：非金融纯 Agent OS 订阅客户数单独汇报。

漏斗：MQL → SQL、SQL → Paid、Trial → Paid。

SLA：Amazon PR 线索 24 小时内首触和下一步，目标 ≥ 95%。

合规：额度、费率、审批承诺事件目标为 0。

价值假设

跨境卖家愿意为“更快看清经营问题、更快得到资金方案、更持续管理增长风险”付费。Agent OS 提供判断与管理，Trio 产品提供资金执行，Referral 降低获客成本。

风险与决策

核心决策：销售以 Agent OS 为主叙事，FastPay / TermPay / MoreFund 作为行动层。

主要风险：客户听不懂、销售过度承诺融资、Amazon 节点延迟、Demo 不够真实。

应对：SG-SKU 管 owner/date，每周报 ARR 与漏斗，Amazon PR 后 24h SLA，额度/费率/审批不得承诺。
